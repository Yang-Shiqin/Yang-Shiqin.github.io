部署在github.io

用personal token的方式验证

前期准备

  • github账号
  • 创建username.github.io仓库

生成personal token

  1. 头像-settings-最下面developer settings-personal access token-tokens(classic)-右上角generate new token
  2. 填写 note, expiration(token过期之后就会认证失败，需要重新激活或者使用新的token), 勾选 repo
  3. 生成token。在离开这个界面后这串代码将不再可见，因此在刷新界面或者离开界面前一定复制好这串代码。

部署

  1. 设置部署。更改 _config.yml 文件最后的 deploy 模块
  2. 部署 hexo clean && hexo deploy 或 hexo d。注意，hexo会把你之前的代码删除覆盖

部署目录和写作目录分开存储

个人网页保存在 .deploy_git 上传到了 <username>.io 的仓库，而最重要的写作源文件所在文件夹 source 并没有上传到github上。推荐新建一个仓库（可设成private）上传 source 文件夹

hexo介绍

hexo常用命令汇总

hexo目录结构

hexo init 后的目录结构

source -> public -> .deploy_git

  • hexo g : 根据 source 更新/生成 public 和 db.json
  • hexo d : 根据 public 更新/生成 .deploy_git
  • hexo clean : 清除 db.json 和 public

_config.yml

全局配置文件，网站的很多信息都在这里配置，诸如网站名称，副标题，描述，作者，语言，主题，部署等等参数。这个文件下面会做较为详细的介绍。

scaffolds

模板文件夹。包含 page，post，draft 三种布局模板，分别对应 页面、要发布的文章、草稿。

布局 生成路径 含义
page source 页面
post source/_posts 文章
draft source/_drafts 草稿
自定义 source/_posts 自定义模板

禁用布局只需在文章的yaml头（front/matter）中设置 layout:false

也可以自定义其他类型的模板文件，只需：

  1. 在此目录下创建 xxx.md 模板文件
  2. 在创建文章时指定 xxx：hexo new xxx <title>

source （重要）

资源文件夹，存放用户资源（如图片，md、html文件等）。除了 _posts ，其他 _ 开头的文件/文件夹都会被忽略。md、html文件会被解析并放到 public 文件夹，其他文件会被拷贝过去

  • source/_posts：存放md源文件
    • md源文件编译成html放到 public 下
    • 每个页面有以该页面命名的文件夹，也在 source/_posts
  • source/categories：分类文件夹，有 index.md，由 hexo new page categories 生成
  • source/tags：由 hexo new page tags 生成的标签文件夹，自动生成 index.md
  • source/_discarded：删除的页面在这

public

hexo g 根据 source 自动编译生成/更新。生成 index.html 也就是我们最后看到的网页

hexo clean 会清除 db.json 和 public

.deploy_git

hexo d 自动编译生成/更新，将 public 提交到github上后生成 .deploy_git，这也是我们上传到github中的内容。内容与 public 差不多。

hexo d 也能生成 db.json 和 public
