基础知识

FFMpeg处理音视频 ==流程== ：
![[Pasted image 20240927141544.png|500]]

  • 输入/输出文件：具有封装格式的文件。如 .mp4、.flv、.avi、.ts 的（视频、音频、字幕）一块的文件或流
    • 格式只和muxer有关
  • 编码数据包：单独的编码后（如压缩）的视频（视频编码数据，如 .h264、.mpeg2、VC-1）、音频（音频编码数据，如 .aac、.mp3、AC-3）、字幕数据
    • 格式只和encoder有关
  • 解码后数据帧：原始数据。视频像素数据，如 .yuv、RGB；音频采样数据，如 .pcm
    • 原始数据和编码数据的差别主要在于体积

录制命令

  • 停止录制：录制过程中按 q
  • ffplay快捷键：
    • q / esc：退出
    • f：全屏
    • p / space：全屏
  • 参数说明：
    • -f x11grab：（用什么采集）输入设备使用 X11 图形服务器进行屏幕抓取
    • -i :0：（从哪采集）捕捉第一个 X11 显示屏（echo $DISPLAY 可以查看设备）
  • 设备devices -f（不同设备的设备编号不同，即 -i 不同）：
    • x11grab：用于捕获 X11 窗口系统中的屏幕内容。可以录制桌面或窗口区域
    • alsa：用于捕获音频，基于 ALSA (Advanced Linux Sound Architecture) 音频系统，设备编号如 hw:0
    • v4l2 (Video4Linux2)：用于捕获视频，通常是从摄像头设备获取视频流，设备编号如 /dev/video0
  • 查看设备编号 -i：
    • ffmpeg -f 某种设备 -list_devices true -i ""
      • x11grab 不支持 -list_devices，因为是直接从x11抓取
      • macOS 上的 avfoundation 和 Windows 上的 dshow 都支持
    • arecord -l：列出所有可用的 ALSA 音频设备，列出 card x ... device y，则设备编号为 hw:x,y
      • ALSA是底层的音频驱动，直接与硬件交互
    • pactl list short sources：列出 PulseAudio 中可用的音频源，包括麦克风、音频输入设备、虚拟设备等
      • PulseAudio 是高层的音频服务器，是ALSA的封装，可以显示虚拟设备、监控设备等
      • 输出说明：
        • 第一列为设备编号
        • s16le：s 是signed，表示有符号；16 是16位；le 是little-endian，小端
        • 2ch：2 channel，双声道
        • 48000Hz：采样频率
        • SUSPENDED：挂起，表示没有音频活动，它们会在有音频数据传输时自动恢复活跃状态
        • 1的 output 是监控输出设备采集，2的 input 是输入设备采集

滤镜命令

简单滤镜（滤镜链） 命令格式，即 -vf 的值：

  • [input_link_lable*] 是该滤镜链的输入连接符号(link label)，可以有多个，表示滤镜的输入
    • 格式为 [<number|name>[:a|v][?]]
      • 如 [0:v] 表示第1个输入的视频流，[0] 表示第1个输入，[outv] 表outv
      • ? 表示该流是可选的，如 [0:v][0:a?] 表示第1个输入的音频可选
  • [output_link_lable*] 是该滤镜链的输出连接符号，可以有多个，表示滤镜的输出
  • filter1=para11:para12,filter2=para2,... 是滤镜链，由多个滤镜组成，用 , 分隔，顺序执行
    • filter* 是滤镜名称，如 crop、rotate，具体取值可以 ffmpeg -filters 查看
    • para* 是滤镜输入参数，= 指定关键字参数，: 分隔，具体可用参数名通过 ffmpeg -h full | less,/<filter>\sAV 或 ffmpeg -h filter=<filter> 查看
    • 示例：
      • crop=in_w-120:120:60:60
        • 参数顺序：w，h，x，y
        • in_w 是输入（上个滤镜输出）图片的宽度，in_w-200 是相对其减少200
      • crop=w=120:h=120:x=60:y=60,rotate=PI/2
        • 显式赋值，可以不用管参数顺序
        • 裁剪后顺时针旋转90度
  • -c:v libx264：使用 H.264 编码视频

ffmpeg开发(cAPI)

版本：ffmpeg-7.0.2

ffmpeg是c写的，一共包含8个库（libxxx）：

  • avcodec：编解码器（最重要的库）
  • avformat：封装格式处理，即流、容器、IO的操作
  • avfilter：滤镜特效处理（最复杂的模块）
  • avdevice：各种设备的输入输出
  • avutil：工具库（大部分库都需要这个库的支持），包括hash器、解码器和工具
  • postproc：后加工
  • swresample：音频采样数据格式转换，混音（通道方向）和重采样
  • swscale：视频像素数据格式转换，图像缩放、颜色空间转换

可能迭代更新，后面不会具体介绍各函数的用法，只介绍抽象功能，具体用法参考以下资料：

  • 官网API：FFmpeg: Main Page
  • 应用代码：github/codeBase/01-study/av/basic_ffmepg

流操作

一个封装格式的文件（format）（mp4等）有很多流/轨（stream）（如多个音频、多个视频，是平行而不相交的），一个流有很多包（packet），一个包由一个或多个帧（frame） 组成（包是压缩的，帧是解码后的）

从 FFmpeg 4.0 开始，av_register_all 逐步被淘汰。新的 FFmpeg API 不再需要显式调用该函数，因为 FFmpeg 会自动在内部初始化（注册 FFmpeg 支持的所有编解码器和复用器等）。

解复用(format->packet)

封装格式数据mp4等->aac等

工作流程：

  1. 打开输入文件（具有封装格式的），获取其context信息：avformat_open_input
  2. 获取所需流（如音频/视频）的index，可用 av_find_best_stream
  3. 初始化包结构以存放读入的packet：av_packet_alloc
  4. 从输入文件读取index流中的packet，并处理
    1. 读packet：av_read_frame
    2. 判断packet所属流的index，是则 处理
    3. 释放packet：av_packet_unref
    4. 继续读入下一packet
  5. 释放资源：av_packet_free，avformat_close_input

处理 所做的事因封装格式而异（需要了解[[音视频基础]]和各种协议），如：

  • 音频
    • MP4 -> AAC：AAC采用ADTS封装格式则需要在每个写入的 pkt->data 前写入ADTS头
  • 视频
    • AVI/MPEG-TS -> H264：直接把 pkt->data 写入文件即可
    • MP4 -> H264：AVVC->Annex-B。需要对所有帧去掉长度前缀，加上start code；对每个IDR帧，帧前加入SPS、PPS帧

解码(format->frame)

封装格式数据mp4等->yuv等

工作流程（增加了解码、裁剪等）：

  1. 打开输入文件（具有封装格式的），获取其context信息(不完整)：avformat_open_input
  2. 获取输入文件详细信息并填入context结构体：avformat_find_stream_info
  3. 获取所需流（如音频/视频）的index，可用 av_find_best_stream
  4. 根据流信息查找编码器并分配上下文：avcodec_find_decoder、avcodec_alloc_context3
  5. 设置解码器参数并打开解码器：avcodec_parameters_to_context、avcodec_open2
  6. 初始化包结构以存放读入的packet：av_packet_alloc
  7. 初始化帧结构以存放读入的frame：
    1. 存放packet解码后的帧：av_frame_alloc
    2. 存放裁剪后的帧：av_frame_alloc，初始化值，分配缓冲区 av_frame_get_buffer
  8. 初始化缩放上下文：sws_getContext
  9. 从输入文件读取index流中的packet，并处理
    1. 读packet：av_read_frame
    2. 判断packet所属流的index，是则 处理 并解码
      1. 发送包到解码器：avcodec_send_packet
      2. 从解码器接收解码完成的帧（重复这步直到读不出来）：avcodec_receive_frame
      3. 裁剪（由于内存对齐，读的数据右边有黑边）：sws_scale
      4. 保存
    3. 释放packet：av_packet_unref
    4. 继续读入下一packet
  10. 释放资源：av_frame_alloc，av_packet_free，avcodec_free_context，avformat_close_input

格式转换/解复用+复用(mp4->flv等)

工作流程（创建格式上下文用于复用，复制所有流信息和流内容，转换时基）：

  1. 打开输入文件（具有封装格式的），获取其context信息(不完整)：avformat_open_input
  2. 获取输入文件详细信息并填入context结构体：avformat_find_stream_info
  3. 初始化输出上下文结构体：avformat_alloc_output_context2
  4. 复制输入流信息到输出上下文：
    1. 遍历输入流（只处理视频、音频、字幕）：ifmt_ctx->nb_streams, in_stream->codecpar->codec_type
    2. 初始化输出上下文的流结构体：avformat_new_stream
    3. 从输入流拷贝流的编解码信息到输出流：avcodec_parameters_copy
  5. 若输出是文件，则打开输出文件到输出上下文结构体：avio_open
  6. 将流头写到输出：avformat_write_header
  7. 初始化包结构以存放读入的packet：av_packet_alloc
  8. 从输入文件读取流中的packet，并处理
    1. 读packet：av_read_frame
    2. 判断packet所属流是否为我们要拷贝的（视频、音频和字幕），是则
      1. 对时间戳等进行时基转换：av_rescale_q_rnd
      2. 将数据包按顺序写入输出：av_interleaved_write_frame
    3. 释放packet：av_packet_unref
    4. 继续读入下一packet
  9. 将流尾写入输出：av_write_trailer
  10. 释放资源：avio_closep，av_packet_free，avformat_free_context，avformat_close_input

裁剪

工作流程：

  1. 解复用和复用的基础上
  2. 查找开始时间对应的帧：av_seek_frame
  3. 保存每个流的开始pts和dts，转换时间戳的时候还要减去其
    • pts：Presentation Time Stamp，展示时间戳（单位为时基）
    • dts：Decoding Time Stamp，解码时间戳
  4. 如果时间戳超过结束时间对应的时间戳则丢弃

编码(frame->packet)

原始数据，如相机采集的数据->编码数据

工作流程：

  1. 打开编码器
    1. 找到编码器：avcodec_find_encoder_by_name
    2. 分配上下文：avcodec_alloc_context3
    3. 设置编码参数（手动），比特率、宽高、帧率等
    4. 打开编码器：avcodec_open2
  2. 创建packet结构体以存放编码后的数据
  3. 创建frame结构体存放原始数据
    1. 初始化frame结构体：av_frame_alloc
    2. 设置frame参数
    3. 申请frame缓存：av_frame_get_buffer
  4. 编码
    1. 接收frame数据（从相机采集）：avcodec_send_frame
    2. 编码到packet并输出：avcodec_receive_packet
  5. 读取剩余packet并输出（因为编码器内部的缓存机制，发送NULL帧后，编码器才会把剩余所有返回）
  6. 添加结束码：mpeg或h264结束码是 0, 0, 1, 0xb7
  7. 释放资源

用的别的库，ffmpeg只是做了一层封装，如h264编码，用到了libh264
