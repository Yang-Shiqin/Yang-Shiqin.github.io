基础篇

[!summary]

  • __new__
  • __init__
  • __del__
  • __repr__
  • __str__
  • __format__
  • __bytes__

__new__ 和 __init__ : 可以改变从一个类建立一个对象的行为，在建立对象的时候传参，则参数会传给二者

  • __new__ : 从一个class建立一个object的过程
  • __init__ : 给存在的object初始化的过程

__init__ 用的更多，初始化对象的时候用到。那什么时候会用到 __new__ 呢（在建立obj前要do sth的时候）

  • singleton class：在建立obj前，判断是否有其他obj被建立了
    • 可能多次调用 __init__
  • 与metaclass有关的内容

__del__ :

  • 类似析构函数，但不一样。
  • 当对象被释放时调用。
  • 不是很好用，因为python的释放机制很复杂，可能是引用为0时，可能是garbage collection时。所以不可控。
  • del 和 __del__ 无关，del 只是让obj少个引用

__repr__ 和 __str__ : 都是返回obj的字符串表示，主要是语义上的不同

  • __repr__ : 更详细的信息
  • __str__ : 人类能理解的，侧重可读性

两个方法都定义的情况下，print(obj) 使用的是 __str__，可用 repr(obj) 和 str(obj) 指定调用。当没有详细和简略版时，只定义 __repr__

__format__ : 使用频率更低，当要以某种格式打印obj时使用

__bytes__ : bytes(obj) 调用

比较篇

rich comparison。

  • 同类的obj比较是从前往后的顺序，如 x comp y 就是 x.__comp__(y)
  • 非同类不满足上面的规则
    • 若y是x的衍生类（继承x），优先使用y的rich comp，如 x < y 就是 y.__gt__(x)；否则优先x

类似重载运算符

[!summary]

  • __eq__ : ==
  • __ne__ : !=
  • __gt__ : >
  • __lt__ : <
  • __ge__ : >=
  • __le__ : <=
  • __hash__
  • __bool__ : if obj

等于不等于：python的对象 __eq__ 比较默认用的 is； __ne__ 若没定义，则默认是 __eq__ 结果的取反

大于小于：

  • __gt__ 等不默认实现，需要自己实现
  • python默认大于小于是一对，所以实现 __gt__ 也能用 <
  • 若实现 __lt__ 则 < 优先调用 __lt__。
  • python不会推测小于等于 __le__ 或大于等于 __ge__，即不会认为他们是小于+等于

哈希 __hash__：

  • 自定义数据结构有默认的hash算法 hash(x)
  • 但若用户定义了数据结构的 __eq__ 方法，则会把默认的 hash给删除。想用就得自定义
    • 因为hash：两个东西相等，则其hash必须相等
    • 改变默认 __eq__ 函数，则hash就不对了
  • __hash__ 要求
    • 必须返回一个整数
    • 对相等的对象，hash值必须相等
    • 官方建议用python提供的hash built-in func，可以把核心属性组成tuple传入

bool __bool__：自定义对象默认为真。定义 __bool__ 返回True/False

属性篇

[!summary]

  • __getattr__
  • __getattribute__
  • __setattr__
  • __delattr__
  • __get__
  • __set__
  • __delete__
  • __slots__

__getattr__ : 定义当取对象不存在的属性时会做什么，即只有在访问不存在属性时才会调用

  • 默认会raise AttributeError。
  • access的attribute会以str的形式传入 __getattr__

__getattribute__ : 名字取得不是很好。只要访问属性就会被调用，即 __getattr__ 也会调用 __getattribute__；另外写的时候要很小心，因为很容易递归

__setattr__ : 写属性时调用

一个神奇的用法：class所有obj共享attribute

__delattr__ : 和 __del__ 不同，obj正常产生和消亡过程中不会被调用，尝试删除属性时(del o.data)才会被调用

打印所有属性 __dir__ : 通过 dir(obj) 调用

  • __dir__ 必须返回sequence

descriptor方法：__set__、__get__、__delete__

一个对象在访问描述器时，如果该属性是一个描述器，则默认属性回调规则会被__set__, __get___, __delete__ 方法所覆盖

Python内部自带的staticmethod, classmethod, property，super等都是描述器，在很多Python库中也都有描述器的身影（例如SQLAlchemy），使用描述器能让你有更高的概率写出优美的代码、更简洁的API，并会加深对Python理解

__slots__ : 不是方法，是特殊变量名。规定class的obj中可以有哪些自定义的attribute（__init__ 中定义的也是自定义的）

类构建篇

[!summary]

  • __init_subclass__
  • __set_name__
  • __class_getitem__
  • __mro_entries__
  • __prepare__
  • __instancecheck__
  • __subclasscheck__

__init_subclass__ : 定义在基类里，当以此基类定义一个衍生类的时候，这个方法会被调用

__set_name__ : 更多的是用在描述器。相当于一个hook。在做log的时候可以使用。用的不是很频繁

强化typing的时候被引入的 : __class_getitem__ 和 __mro_entries__

  • __class_getitem__ : cls[item] 会调用
    • __getitem__ : obj[item] 会调用
    • 如 list[int] 就是用 __class_getitem__ 实现的
    • __mro_entries__ : 帮忙找基类，返回tuple（obj告诉你哪些是你的基类

metaclass相关的魔术方法 : __prepare__、__instancecheck__、__subclasscheck__

  • __instancecheck__ : 对应 isinstance()
  • __subclasscheck__ : 对应 issubclass()

运算篇

[!summary]

  • __add__
  • __sub__
  • __mul__
  • __matmul__ : @
  • __truediv__ : /
  • __floordiv__ : //
  • __mod__
  • __divmod__ : divmod()
  • __pow__ : **
  • __lshift__ : <<
  • __rshift__
  • __and__ : &
  • __xor__
  • __or__ : |
  • __r运算符__ : 运算符的r版本
  • __i运算符__ : 运算符= 运算符的inplace版本，本地操作
  • __neg__ : 单目 -
  • __pos__ : 单目 +
  • __abs__ : abs()
  • __invert__ : ~
  • __complex__ : complex()
  • __int__ : int()
  • __float__ : float()
  • __index__ : [obj] 时把 __index__ 返回值放入 []；int/float/complex 没定义则默认 index
  • __round__ : 四舍五入
  • __trunc__ : 向零取整
  • __floor__ : 向负无穷取整
  • __ceil__ : 向正无穷取整
  • 重载和多态，__运算符__(self, other)，other不一定是同类型对象，可以通过判断对象类型规定不同运算
  • a 运算符 b(a.__运算符__(b)) 和 b 运算符 a 不一样
  • 当遇到 other 运算符 self 且other不知道这个操作时，调用self的 __r运算符__

模拟篇

[!summary]

  • __call__
  • __len__
  • __reversed__
  • __contains__ : in
  • __iter__ : 迭代器

模拟函数

  • __call__ : 让对象可以像函数一样被调用

模拟容器 (如list(sequence), dict(mapping))

  • __len__ : 返回长度 len(obj)。若没定义 __bool__ 则 if obj: 会找 __len__，为0则False
  • __length_hint__ : 返回估计的长度，operator.length_hint(obj)，很少用
  • __getitem__ : [] 取值
  • __setitem__ : [] 赋值
  • __delitem__ : del x[0]

__missing__ : subclass才有用。找key但找不着调用，返回找不到该返回什么

context type : with xx，with后的就是context。定义一个context要定义其 __enter__ 和 __exit__

  • __enter__ : 进context干啥
  • __exit__ : 出context干啥
    • 参数 exc_type, exc_value, traceback 是在出现exception时使用
      • exc_type, 异常class
      • exc_value, 异常值
      • traceback, obj
    • 出错但context内容依然会被执行，很适合拿来释放资源
